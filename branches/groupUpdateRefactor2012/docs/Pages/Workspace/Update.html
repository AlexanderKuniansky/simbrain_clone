Update Types

Simbrain update is complex.  Under the hood, there is an updator that manages workspace updates.  The updator allows you to configure the system in various ways, and automatically takes advantage of multiple processors.

The default update updates each component, then couplings.  This in turn can happen in several ways.

Buffered: update order does not matter. The default sequence of events is:
    - update component 1
    - update component 2, ...
    - update component n
    - update couplings (uses a buffer so that update order does not matter)

Priority: update order does matter.  When you need things updated in a particular sequence.

One thing you can do is customize how the workspace is updated.   This can  be done by running a script which installs a custom "updater".    I have added an example which shows how this is done in the  scriptmenu.   If you run the script "customUpdateExample.bsh", you will notice that the network's synapses are randomized every time the workspace is updated.


Updator tab: Can specify one or more threads.  It defaults to the number of available processors which can be changed.

Custom update: users can create their own updators.  In a script, just create an instance of the ... and call workspace.setUpdator.  See scripting.
