Hopfield Network

A (discrete) hopfield network is a collection of binary neurons which are fully interconnected but without self-connections. These are part of a subnetwork because they must be updated differently than standard networks which are updated synchronously. Hopfield networks must be updated asynchronously in order to be guaranteed to converge on stable states (you can easily create a synchronous version of a Hopfield network using the standard network, in which you can frequently observe unstable behavior).

Hopfield networks are trained via the Hebb rule. However the synapses in a Hopfield network are not Hebbian, but clamped (the synapse values are set by the hopfield network itself). To train a hopfield network set the neurons to some desired level and select "train" from subnetwork tab context menu.

Hopfield networks also contain a number of convenience methods. They are initialized to be made of binary neurons which are appropriately connected. When they are randomized the weights are kept symmetrical.

To create a continuous Hopfield network use Additive neurons. These can be connected appropriately and trained by using Hebbian synapses. The user then clamps all neurons, iterates to train the synapses, then clamps all weights. On clamping, see toolbar.

Initialization

Hopfield networks are initialized with some number of units, and are by default laid out as a grid. They are fully interconnected with no self connections.

 

Parameters

Update Order: This can be set to random or sequential. If set to random, the neuronns are updated in random order. This is the standard assumption of Hopfield networks. if sequential is used, neurons are updated in the same sequence each time, making it possible to reproduce chains of behavior.

Randomize

Randomizes so that all connections remain symmetric.

Train

Trains the hopfield network using the Hebb rule to learn the current pattern of activity across its nodes.
