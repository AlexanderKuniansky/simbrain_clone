Network Update Methods

Each time the step button is pressed in the network-panel, the network goes through an iteration. As a result of this iteration, the neuron and sub-network activations and the synapse strengths get updated. Since some of the neurons and sub-networks in your network could be acting as a source of input for other neurons and sub-networks, the activation values of the individual neurons and sub-networks will depend on the order in which they are calculated. Further, since in most cases, the synapse weights are updated based on the current activation values of the sending and receiving neurons, the updated weight value will also depend upon the order in which the neuron activation values were computed. By default, the neuron activation order is completely random. However, there are two different ways in which you could enfore ordering on the neuron activation sequence, as explained below.

Open the network preference dialog by right clicking within your network window. The Logic tab of the dialog will show a property named Network Update Method. This property can take one of the three values:
1. Standard
2. Priority based
3. Script based

Standard update method is the default, in which, the user has no control over the order in which the neurons and subnetworks will be updated.

If you want some of the neurons and subnetworks to be updated before others, you could use the Priority Based update method. Each neuron and sub-network has a property named as the Update priority. For a neuron, this property is listed in the neuron property dialog. For a sub-network, this property can be set by right clicking on it. By default, this is set to 0 for all elements. You can change it to any other integer value. If you set the Network Update Method as Priority Based, the activation values for the elements will be calculated in the increasing order of their Update priority value.

If you write your own script for updating the network elements, you will be responsible for updating all properties of your network, including the neuron and sub-network activation values, synapse strengths, updating the UI and so on. This is tedious, but gives you the maximum degree of felxibility and control.
