Network Component

The network component is used to build neural networks which simulate circuits in an organism's brain, or, more abstractly, computation using neuron-like elements.

Networks are built by creating neurons and connecting them with synapses. They are run by pressing the "play" or "synapse" buttons. There are many varieties of neurons and synapses, each with their rules and parameters. By combining these together you can create indefinitely many different kinds of networks. There are also special types of networks called "subnetworks" which can be added. Different subnetworks can be linked together, leading to even more complex and powerful siulations.


Example

Shown below is a simple neural network. The colored circles with numbers in the middle represent simplified neurons or "nodes." The lines between nodes represent synaptic connections between neurons which direct the flow of activity in the network. The smaller blue and red discs at the ends of these lines represent synaptic connections.

 

Activation flows along the connections in the direction of the discs. Activation flows from left to right in the network above. Since this network is not connected to the world component (it has no input or output nodes), to experiment with this network a user would normally adjust the activation of the left-most nodes by hand (see manual adjustment of neurons and weights), and then use the single step key to iterate the network.

 

Visual Conventions

Red nodes represent "excited" neurons.
White nodes represent inactive neurons
Blue nodes represent "inhibited" neurons.

The numbers inside the neurons represent their "activation level" (we can think of the absolute value of this number as representing the rate of a firing of a neuron).

Red weights represent excitatory synaptic connections.
Blue weights represent inhibitory synaptic connections.

The diameter of the discs represent their strength (all of the weights above have the same absolute value).

 

Running the Network

When Simbrain is first opened nothing happens. A sample network sits passively before the user. When the user "runs" the network it comes to life--an update algorithm is repeatedly called, which is visible in the form of spreading activity in network's nodes and (if learning is activated) changing weight sizes. The network can be run in any of the following ways:

Run the network

Press the play button in the tool-bar to repeatedly iterate the network. Note that the image on the button becomes that of a "stop" button . Pressing this button will stop the network. Thus the play/stop button acts as a toggle switch.

Step the network

Press the step button or press "space" or "s" in the keyboard to iterate the network a single time. Sometimes repeatedly hitting the space bar is useful way to test a network. (Note that "s" is special in that it overrides the network-world interaction button and only updates the network).

World-based iteration

The network will also run if an appropriate action is taken in one of the open and coupled worlds (assuming network-world interaction is set to both-ways or world-to-network). For example, moving a creature in an OdorWorld will update all connected networks.
