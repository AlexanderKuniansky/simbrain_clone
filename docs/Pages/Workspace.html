Workspace

The Simbrain workspace is the framework within which a simulation is run. It contains components which are coupled to one another, and tools for managing these interactions.  In the GUI, the workspace is visible as the Simbrain desktop.

The workspace  is responsible for (1) maintaining a list of components, (2) a list of couplings and a coupling manager for managing interactions between componetns, (3) an updator for managing simulation update.

Most windows in Simbrain and workspace components, which are wrappers around other objects, which are designed to exist in a workspace. Couplings tie these components together via attributes.   For example, a simulation might involve coupling neurons whose neurons are producers to a bar chart whose bars are consumers (to see this, try bar chart test in the script menu).


Components [move this]


Adding components: To add components press the relevant button on the toolbar, or use the Insert menu.

Saving / Opening: Use the save, open buttons and the File menu.

All open components are shown in the Components tab of the bottom information pane.

Running a simulation [move this]

 

Press the iterate and play buttons in the main toolbar.

In a terminal component, or in the terminal tab of the bottom information pane, use workspace.iterate(N), where N is the number of iterations to iterate.

Overview

The main things you need to do to make a simulation are:

1) Add components
- Use the buttons
- Add from a script

2) Connect together components
- Use the coupling manager
- Use the menus (single or group)
- Make them in a program script (and then, for GUI, save)

3) Run the simulation
- Use the built in method (buffering based)
- Create your own update method
