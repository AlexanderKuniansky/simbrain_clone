Workspace

The Simbrain workspace is the framework within which a simulation is run. It contains components which are coupled to one another, and tools for managing these interactions.

In the GUI, the workspace is visible as the Simbrain desktop.

The overall design is (1) a list of components, (2) a list of couplings and a coupling manager for managing interactions between componetns, (3) an updator for managing simulation update.

Most windows in Simbrain and workspace components, which are wrappers around other objects, which are designed to exist in a workspace. For you the user, this mainly means attributes (for couplings). Couplings tie these objects together via attributes. Attributes are the "hooks" or "anchor points" that make up couplings. Attributes produce data, and consume data. A coupling consists of one producer and one consumer. For example, a simulation might involve coupling neurons whose neurons are producers to a bar chart whose bars are consumers (to see this, try bar chart test in the script menu).

Components

 

Adding components: To add components press the relevant button on the toolbar, or use the Insert menu.

Saving / Opening: Use the save, open buttons and the File menu.

All open components are shown in the Components tab of the bottom information pane.

Adding Couplings

 

A coupling is a link from an object in one component to an object in another component (or the same component). A producer produces values that are read by a consumer.

A list of all current couplings is shown in the Coupligns tab of the bottom information pane.

To create couplings:

  1. Use the coupling manager window.
  2. Use the general coupling menu item
  3. Use the coupling popup menus

Running a simulation

 

Press the iterate and play buttons in the main toolbar.

In a terminal component, or in the terminal tab of the bottom information pane, use workspace.iterate(N), where N is the number of iterations to iterate.
