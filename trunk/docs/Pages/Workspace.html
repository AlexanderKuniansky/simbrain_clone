Workspace

The Simbrain workspace is the framework within which a simulation is run. It contains components which are coupled to one another, and tools for managing these interactions.

In the GUI, the workspace is visible as the Simbrain desktop.

The overall design is (1) a list of components, (2) a list of couplings and a coupling manager for managing interactions between componetns, (3) an updator for managing simulation update.

Most windows in Simbrain and workspace components, which are wrappers around other objects, which are designed to exist in a workspace. For you the user, this mainly means attributes (for couplings). Couplings tie these objects together via attributes. Attributes are the "hooks" or "anchor points" that make up couplings. There are two kinds, producing and consuming, which spit out and gobble up numbers (currently; may be abstracted to more general datatypes)

The workspace can be thought of as a bunch of things that spit out numbers and read them in. Making a simulation amounts to making number spitter-outers and number-readers.

For example, a simulation might involve coupling neurons whose neurons are producers to a bar chart whose bars are consumers (to see this, try bar chart test in the script menu).

Building a Simulation

The main things you need to do to make a simulation are:

1) Add components
- Use the buttons
- Add from a script

2) Connect together components
- Use the coupling manager
- Use the menus (single or group)
- Make them in a program script (and then, for GUI, save)

3) Run the simulation
- Use the built in method (buffering based)
- Create your own update method

Desktop Interface Components

Bottom Tabs: provide information on the state of the workspace.

Components: list components and allows you to turn them on and off
Couplings: list current couplings
Terminal: interact with workspace directly using beanshell commands (essentially realtime java)
Updator: status of simbrain updator
